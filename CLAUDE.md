# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A single-file static web app (`index.html`) for creating and printing jukebox record labels. No build step, no dependencies, no framework — everything lives in one HTML file with inline CSS and JavaScript.

## Development

Open `index.html` directly in a browser. No server required.

To preview print layout without wasting paper, use the browser's print preview (`Cmd+P` on Mac).

## Architecture

The entire app is `index.html` — HTML structure, `<style>` block, and `<script>` block all in one file.

### Label rendering

Each label is an SVG generated by `makeLabelSvg(id)`. The SVG uses `fill="transparent"` on its text elements — visible text is rendered by absolutely-positioned `<input>` elements overlaid on the SVG via `.label-overlays`. The `sync(id)` function mirrors input values into the SVG text nodes so they appear when printing (since the overlay inputs are hidden in print mode).

### Print layout

The `@media print` block switches the grid to 2 columns and applies precise `@page` margins calculated to fit exactly 2×8 labels per A4 sheet at 78mm label width. The SVG viewBox is `"-10 -10 241.1 91.86"` — the 10pt padding on each side provides space for crop marks.

### Persistence

Labels are saved to `localStorage` under key `jukebox-labels-v1` as a JSON array of `{ta, artist, tb}` objects on every input event. Restored via `loadFromStorage()` on page load.

### Remove button safety

`removeCard(id)` requires two clicks: the first arms the button (shows `✕?`, sets `data-armed`), the second confirms deletion. The armed state auto-clears after 2 seconds.

### Input fields

Three inputs per label: `.ta` (A-side track, italic), `.artist` (bold, center), `.tb` (B-side track, italic). Font sizes use container query units (`cqw`) to scale with label width. Container queries are disabled in print (`container-type: normal`) since sizing is handled by the fixed SVG dimensions.
